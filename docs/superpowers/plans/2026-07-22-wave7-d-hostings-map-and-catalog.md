# Волна 7, План D — хостинги: границы, клик по стране, поиск, карточки, канал

> Пункты пользователя 7 и 8. Один раздел, пять независимых улучшений — поэтому один план с пятью
> фазами, каждая отгружаема сама по себе.

## Разведка (проверено по коду)

**Карта** — `frontend/src/components/hostings/HostingsMap.tsx`:

- Топология бандлится офлайн: `world-atlas/countries-110m.json` → `feature(...).features` один раз в
  module-scope (`:15-20`). Комментарий объясняет, почему массив, а не объект-топология.
- Границы: `<Geography fill="var(--bg3)" stroke="var(--line-soft)" strokeWidth={0.4}>` (`:120-126`).
  `--line-soft` — самый бледный токен линии, а `0.4` при `geoEqualEarth`/`scale:155` даёт
  субпиксельную линию. При зуме `ZoomableGroup` масштабирует **весь SVG**, включая обводку, поэтому
  на большом зуме границы наоборот жирнеют — но на обзорном виде их фактически нет. Это и есть
  «нечёткие границы».
- `Geography` **не кликабелен** — обработчика нет вовсе; кликаются только маркеры (`:133`).
- Континенты: `enabled: Set<ContinentKey>` (`:33`), фильтр `visible` (`:63-66`), тоглы + «приблизить»
  (`:91-109`). Именно это пользователь просит **заменить** поиском.
- Попап выбранного маркера (`:169-190`) показывает хостинг, город с флагом (`fi fi-<cc>`, `:180`),
  минимальную цену и заметку — то есть кусок нужных данных уже есть, но привязан к точке, а не к стране.

**Каталог** — `frontend/src/components/hostings/HostingsCatalog.tsx`:

- Карточка (`:62`) — `<div className="card p-4 …">` без `onClick`; интерактивны только «карандаш»
  (`:74`) и «корзина» (`:75`). Полного просмотра нет.
- Тариф в форме: имя, `specs` (свободный текст), цена, валюта, период (`:174-189`). Ширины канала нет.

**Модель** — `backend/app/models/hostings.py:8-13`:

```python
class Tariff(BaseModel):
    name: str = ""; specs: str = ""
    price: float = Field(default=0, ge=0)
    currency: str = "USD"; period: str = "mo"
```

Поля под канал нет ни в `Tariff`, ни в `Location` (`:16-21`).

**Чего в коде нет, вопреки плану Волны 4:** geo-resolve эндпоинта. Резолв координат — клиентский
(`hostings/geo.ts::resolveCoords`, зовётся на `:47` и из формы `:217`). Учитывать: «страна» у локации
берётся из `country_code`, который может быть пустым.

## Премисса, которую надо проверить ПЕРВЫМ шагом

Клик по стране требует связать geo-фичу с нашим `country_code` (alpha-2). У `world-atlas`
`countries-110m` фича несёт **числовой** ISO-код в `id` (`"528"` для Нидерландов) и английское имя в
`properties.name`. Это надо подтвердить на бандленном файле до написания UI:

```
node -e "const t=require('world-atlas/countries-110m.json');const {feature}=require('topojson-client');
const f=feature(t,t.objects.countries).features;console.log(f.length, JSON.stringify(f[0].id), Object.keys(f[0].properties))"
```

(Node на машине нет — гонять в контейнере `ni-frontend-test`, см. «Верификация».)

Если `id` окажется не числовым ISO — **остановиться и пересогласовать Ф3**; Ф1/Ф2/Ф4/Ф5 от этого не
зависят.

## Развилки

**Р1. Чёткость границ.**
- **(а, по умолчанию)** оставить `countries-110m`, поднять контраст: `stroke="var(--line)"`,
  `strokeWidth` ≈ 0.6 и **`vectorEffect="non-scaling-stroke"`** — обводка перестаёт масштабироваться
  зумом, то есть остаётся одинаково чёткой и на обзоре, и на приближении. Бандл не растёт.
- (б) перейти на `countries-50m.json` — геометрия детальнее, но файл примерно вчетверо больше.
  План F Волны 6 уже намерил **JS 1.43 МБ**, и раздел хостингов тянет `react-simple-maps` +
  `topojson-client` + `motion`. Утяжелять его ради обводки — работать против собственного
  перф-плана. Если после (а) детализации всё равно мало, `50m` подключать **только** ленивым
  чанком этого раздела.

**Р2. Источник списка стран для клика.** Нужна таблица `ISO-numeric → alpha-2`.
- **(а, по умолчанию)** маленькая таблица в `hostings/geo.ts` рядом с существующими данными региона.
  Полный набор не нужен: достаточно кодов, встречающихся у наших локаций, плюс общий фолбэк «страна
  без хостингов — клик игнорируется».
- (б) матчинг по `properties.name` — английские имена, расходящиеся с нашим `COUNTRIES`
  («Czechia»/«Czech Republic», «United States of America»). Хрупко, отвергаю.

**Р3. Что показывать по клику на страну.** Пользователь перечислил: название, город (с флагом),
сетевой канал(ы), цена и тариф. Значит панель группирует по **хостингу**, внутри — строки локаций и
тарифов этой страны. По умолчанию — **боковая панель**, а не модалка: карта остаётся видна, можно
кликать соседние страны подряд. Существующий попап маркера (`:169-190`) при этом остаётся для точки.

**Р4. Что ищет поиск.** Одна строка, без синтаксиса: подстрока по имени хостинга, городу, стране
(и alpha-2, и русскому имени — переиспользовать `resolveCountryCode` из Плана A), названию тарифа,
`specs`, каналу; для цены — числовое сравнение при вводе вида `<20`, `>5`, `10-30`, иначе цена
игнорируется. Результат: фильтрует маркеры **и** подсвечивает совпавшие страны.

## Риски

- **`country_code` пустой.** У локации это необязательное поле (`models/hostings.py:18`). Такие
  локации не попадут ни в клик по стране, ни в фильтр по стране — показать это в UI («без страны»),
  а не терять молча.
- **Кликабельные страны и производительность.** 177 путей с обработчиком + подсветка при вводе в
  поиск: подсветку считать по `Set<alpha2>` из `useMemo`, а не искать по массиву внутри рендера
  каждой фичи.
- **Регресс перф-базы.** После Ф2-Ф4 переснять сцену хостингов харнессом `tests/e2e/perf-probe.mjs`
  (Волна 6, План F Ф1) — раздел и так самый тяжёлый по зависимостям.
- **`specs` vs `bandwidth`.** У части хостингов канал уже вписан в `specs` руками. Миграцию не делаем
  (данные пользовательские, парсить свободный текст — гадание); поиск ищет по обоим полям, так что
  старые записи находятся.

## Критерий готовности

1. Границы стран различимы на обзорном виде в обеих темах и не расплываются при зуме.
2. Клик по стране открывает панель со списком хостингов: название, город с флагом, канал, цена, тариф.
3. Тоглов континентов нет; поиск находит по стране, городу, хостингу, тарифу, каналу и цене.
4. Клик по карточке хостинга открывает полные данные.
5. В тарифе есть ширина канала; старые документы читаются без миграции.
