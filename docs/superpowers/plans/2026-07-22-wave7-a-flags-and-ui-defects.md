# Волна 7, План A — флаги стран и точечные дефекты интерфейса

> Пункты пользователя 1 и 3. Оба — визуальные дефекты с уже понятной причиной, поэтому план короткий
> и полностью проверяемый юнит-тестами (Playwright не нужен).

## Разведка (проверено по коду)

### Пункт 1 — «anscii коды стран не конвертируются в флаг»

Скриншот показывает строку узла с синим глобусом и текстом `NL Нидерланды 🚀`, где `NL` нарисован
мелкими буквами в рамке. Это **не** сырой ASCII-текст — это ровно то, как Windows рисует пару
regional-indicator `🇳🇱`, для которой в системном шрифте нет глиф-лигатуры флага.

Причина задокументирована в самом коде, который её и создаёт:

- `frontend/src/utils/format.ts:16-27` — `getFlagEmoji` строит пару regional-indicator (`127397 + charCode`).
  Комментарий на `:11-14` прямо предупреждает: *«some Windows builds don't render regional-indicator pairs
  as flags and show the two letters instead… swap this for an SVG icon set (e.g. flag-icons) at the call
  sites»*. Мы этого свопа так и не сделали на дэшборде.
- `frontend/src/components/Dashboard.tsx:48-57` — `flagFor(group)` возвращает **строку-эмодзи**, а не код.
  Порядок попыток: встроенный флаг в строке → строка целиком из 2 букв → поиск по `COUNTRIES`.
- `frontend/src/components/Dashboard.tsx:184` и `:480` — результат рендерится как текст:
  `<span className="text-lg leading-none">{flag}</span>`.

**Второй, независимый дефект** в той же функции: поиск по названию идёт по `COUNTRIES`, где имена
**английские** (`frontend/src/components/CountrySelect.tsx:52` → `{ code: "NL", name: "Netherlands" }`).
Подписка отдаёт `groupName` по-русски («Нидерланды»), поэтому `flagFor` не находит совпадения и
возвращает `"🌐"` — глобус со скриншота. То есть на строке видно **сразу оба** бага: глобус вместо
флага группы (нерезолвнутое русское имя) и буквы `NL` внутри имени узла (эмодзи-флаг из ремарки).

Рабочий образец уже есть в репозитории: `CountrySelect.tsx:76-89` — `FlagChip`, SVG из набора
`flag-icons` (`fi fi-<cc>`), с `Globe`-фолбэком для `XX`. Он же используется в
`hostings/HostingsMap.tsx:180` и `hostings/HostingsCatalog.tsx:19`. Дэшборд — единственное место,
которое осталось на эмодзи (плюс `infra/InfraProviders.tsx:78`).

### Пункт 3 — «Деплой нод» белым в светлой теме

`frontend/src/components/DeployDashboard.tsx:132` — `className="text-base font-semibold text-white"`.
Хардкод из дотемовой эпохи: `text-white` не зависит от `data-theme`, поэтому в светлой теме заголовок
белый по белому. Строка `:133` рядом — `text-gray-500`, тоже хардкод.

Замер: `text-white|text-gray-\d|bg-gray-\d|text-black` встречается в `frontend/src` **26 раз в 2 файлах** —
`DeployDashboard.tsx` (3) и `auth/AuthScreen.tsx` (23). `AuthScreen` — намеренно тёмный полноэкранный
гейт до выбора темы, его **не трогаем**. То есть весь долг по хардкоду сводится к трём строкам.

## Развилки

**Р1. Русские названия стран.** `COUNTRIES` — англоязычный список, зеркалящий пикер панели Remnawave
(`CountrySelect.tsx:4-6`), и он же питает форму деплоя. Варианты:
- **(а, по умолчанию)** отдельная таблица алиасов `ru → alpha-2` в `utils/countryAliases.ts`, `COUNTRIES`
  не трогаем. Пикер деплоя остаётся зеркалом панели, резолв групп получает свой словарь.
- (б) добавить поле `nameRu` в `COUNTRIES` — единый источник, но пикер начинает расходиться с панелью,
  а список станет шире зоны ответственности.

**Р2. Флаг внутри имени узла.** Ремарка подписки часто несёт эмодзи-флаг (`🇳🇱 Нидерланды 🚀`).
- **(а, по умолчанию)** вырезать пару regional-indicator из отображаемого имени и нарисовать её как
  `FlagChip`, если у строки-группы флага не нашлось. Имя становится чище, флаг — везде одинаковым SVG.
- (б) оставить имя как есть — тогда на Windows в имени так и будут мелкие буквы.

## Фазы

### Ф1 — `FlagChip` в общий модуль + `flagFor` возвращает код → verify: `npm test` + `tsc --noEmit`

- Вынести `FlagChip` из `CountrySelect.tsx:76-89` в `frontend/src/components/common/FlagChip.tsx`
  без изменения поведения; `CountrySelect` импортирует оттуда (не дублировать — иначе разъедутся размеры).
- `frontend/src/utils/countryAliases.ts` (NEW) — `ru → alpha-2` для всех 63 кодов из `COUNTRIES`
  плюс частые разговорные формы («США», «Англия», «Голландия», «Эмираты»). Экспортировать
  **чистую** `resolveCountryCode(group: string): string` со порядком: встроенный эмодзи-флаг →
  ровно 2 латинские буквы → английское имя из `COUNTRIES` → русский алиас → `""`.
- `Dashboard.tsx` — `flagFor` возвращает **код** (`resolveCountryCode`), а `:184`/`:480` рендерят
  `<FlagChip code={cc} />`. Пустой код → `Globe`-фолбэк внутри чипа.
- Тест `frontend/src/utils/countryAliases.test.ts` (NEW): `"Нидерланды"→"NL"`, `"NL"→"NL"`,
  `"🇳🇱 Нидерланды"→"NL"`, `"Netherlands"→"NL"`, `"Марс"→""`. Отдельным кейсом — что **каждый** код из
  `COUNTRIES` (кроме `XX`) имеет русский алиас, иначе таблица тихо отстанет при расширении списка.
- **verify:** `docker run --rm -v <repo>/frontend:/w -w /w ni-frontend-test npx vitest run` (Node на
  машине нет — см. §«Верификация» ниже) и `npx --no-install tsc --noEmit`.

### Ф2 — флаг из имени узла → verify: `npm test`

- `Dashboard.tsx` — хелпер `splitFlag(name): {cc, rest}` поверх существующего `extractFlag` (`:37-44`):
  вернуть alpha-2 (обратное преобразование `codePoint - 127397`) и имя без этой пары + лишних пробелов.
- `NodeRow` рисует чип узла: код из имени, если он есть; иначе код группы. Текст — `rest`.
- Тест в `frontend/tests/Dashboard.test.mjs` (существующий framework-free скрипт): `splitFlag` на
  `"🇳🇱 Нидерланды 🚀"` даёт `{cc:"NL", rest:"Нидерланды 🚀"}`; ракета (не regional-indicator) не съедается.

### Ф3 — три хардкод-цвета в `DeployDashboard` → verify: `npm test` + гейт

- `DeployDashboard.tsx:132` → `text-[var(--t-hi)]`, `:133` → `text-[var(--t-faint)]`, третье вхождение
  привести к соседям по файлу.
- Расширить `frontend/src/theme/contrast.test.ts` (из Волны 6, План F Ф1) **новым** кейсом: обход
  `src/**/*.tsx` регуляркой на `text-white|text-black|text-gray-\d{2,3}|bg-gray-\d{2,3}` с
  явным allow-list `["auth/AuthScreen.tsx"]`. Гейт ловит возврат хардкода в любом файле, а не только в
  этих трёх строках.
- **Проверка на разрушение:** временно вернуть `text-white` в `DeployDashboard.tsx` — тест обязан
  покраснеть. Без этого шага фаза не считается выполненной (в Волне 6 ровно так вскрылось фиктивное
  покрытие навигации).

## Риски

- **Набор `flag-icons` не покрывает `XX` и несуществующие коды** — `FlagChip` уже отдаёт `Globe`, но
  проверить, что `fi fi-<мусор>` не рисует пустой прямоугольник с рамкой (`boxShadow` в `:85`).
- **Алиасы — вечный хвост.** Провайдер может назвать группу «🇳🇱 NL-Amsterdam-01». Порядок резолва
  (эмодзи → код → имя) это покрывает, но тест на такой смешанный кейс нужен явно.
- `getFlagEmoji` остаётся в `utils/format.ts` ради `InfraProviders.tsx:78`. **Не удалять** и не
  переводить инфра-биллинг в этой волне — чужая фаза, правило «хирургических правок».

## Верификация (окружение)

Node на машине разработки нет. Фронтенд-проверки идут в Docker:
`docker build -f frontend/Dockerfile --target builder -t ni-frontend-test frontend`, затем
`docker run --rm -v <abs>/frontend:/w -w /w ni-frontend-test npx vitest run`. В Git Bash монтирование
требует `MSYS_NO_PATHCONV=1`.

## Критерий готовности

1. На дэшборде (обе вкладки) флаг — SVG-чип, одинаковый с пикером страны в форме деплоя.
2. Группа с русским названием страны показывает флаг, а не глобус.
3. Заголовок «Деплой нод» читается в светлой теме; гейт хардкод-цветов красный при откате правки.
