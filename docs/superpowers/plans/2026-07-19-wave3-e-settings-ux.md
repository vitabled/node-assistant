# Волна 3 · План E — UX: сворачиваемый терминал SSL, крупные графики, перенос ИИ-чата, проверка соединения

> Пункты: 3a (сворачиваемый терминал в «Управление SSL» + Домены на его месте), 6a (крупные графики «Загрузка
> нод во времени»), 11a (перенос ИИ-чата после «Автоматизации»), 11b («Проверить соединение» по полям формы).
> Пункт 11c (перенос «Оптимизации ОС») — **ОТМЕНЁН**, не делаем.
> Затрагивает: `CertsForm.tsx`/`DomainsPanel.tsx`/`App.tsx` (certs-таб), `stats/UsersStats.tsx`,
> `settings/AiChat.tsx`, `Sidebar.tsx`, `Settings.tsx`, `api/settings.py`.

## Контекст (как есть)

- **«Управление SSL»** (tab `certs`): рендерит `CertsForm` (деплой серта) + `DomainsPanel` (домены аккаунта +
  скачивание). Стрим деплоя серта — задача через `useTaskStream`, вывод терминала (`TerminalOutput.tsx`).
- **«Загрузка нод во времени»** — виджет в `stats/UsersStats.tsx` (a): сейчас мелкие спарклайны (см. фото
  пользователя — тонкие линии без осей/подписей).
- **ИИ-чат** — `settings/AiChat.tsx`, живёт во вкладке Settings→«MCP» (под MCP-конфигом). Стрим ndjson.
- **«Автоматизация»** (tab `automation` → `RuleBuilder`) — sidebar-группа; ИИ-чат надо поставить сразу после неё.
- **Remnawave «Проверить соединение»** — `POST /api/settings/remnawave/check` (`api/settings.py`); сейчас
  проверяет **сохранённые** настройки, а не введённые в поля (`Settings.tsx` Remnawave-вкладка).

## Развилки (закреплены)

- 3a: терминал сворачиваемый, по умолчанию свёрнут, на его месте Домены; при старте новой задачи — авто-разворот.
- 6a: крупный линейный график (ось Y, сетка, ховер-тултип, «сейчас/пик»).
- 11a: ИИ-чат — топ-левел пункт сайдбара сразу после «Автоматизации»; MCP-конфиг остаётся в Настройках.
- 11b: «Проверить соединение» проверяет значения из полей формы (в теле запроса), без сохранения.

## Стратегия

Небольшие независимые UI-правки — можно делать в любом порядке.

---

### Ф1 — «Управление SSL»: сворачиваемый терминал + Домены → verify: preview

- В certs-табе (`App.tsx` / `CertsForm`+`DomainsPanel` контейнер): обернуть область вывода терминала в
  `Collapsible` (переиспользовать компонент из `DeployForm`), по умолчанию **свёрнут**; освободившееся место
  занимает `DomainsPanel`. При старте новой задачи деплоя серта (`deployCert`) — авто-разворот терминала
  (`setState`), по завершении можно оставить развёрнутым до ручного сворачивания.
- verify: preview — терминал свёрнут, Домены на месте; запуск деплоя серта разворачивает терминал и стримит лог.

---

### Ф2 — Крупный график «Загрузка нод во времени» → verify: preview + скриншот

- `stats/UsersStats.tsx` виджет (a): перерисовать спарклайны в **полноразмерный линейный график** (inline-SVG,
  без сторонних либ — CSP self-contained, как остальные виджеты): ось Y (usersOnline), сетка, ховер-тултип со
  значением+временем, подписи «сейчас N / пик M» рядом с легендой. Несколько нод — мультилиния с легендой/
  цветами (палитра как в других графиках). Сохранить шестерёнку time-window (24h/7d/30d).
- verify: preview + скриншот — график крупный, читаемый, тултип работает.

---

### Ф3 — Перенос ИИ-чата → verify: tsc + preview

- Вынести `settings/AiChat.tsx` из вкладки Settings→«MCP» в **топ-левел раздел сайдбара** сразу после
  «Автоматизации» (новый `Tab`, напр. `assistant`; `Sidebar.tsx` порядок + `CRUMB` + `App.tsx` роут). MCP-конфиг
  (`settings/McpTab.tsx`) и конфиг ИИ (`AiConfig`) остаются в Настройках; на новом экране — только чат (конфиг
  можно продублировать ссылкой «настроить в Настройках» или компактным блоком).
- Активная вкладка персистится (`tabKey`, `ni_tab_<accountId>`) — новый tab добавить в валидные `CRUMB`-ключи.
- verify: `tsc`, preview — чат открывается как отдельный раздел после «Автоматизации».

---

### Ф4 — «Проверить соединение» по полям формы → verify: pytest + preview

- `api/settings.py` `POST /api/settings/remnawave/check`: принимать **конфиг в теле запроса**
  (`{panel_url, api_token}` из полей формы) и проверять именно их, ничего не сохраняя. Если тело пустое —
  фолбэк на сохранённые (обратная совместимость).
- `Settings.tsx` Remnawave-вкладка: кнопка «Проверить соединение» шлёт текущие значения полей (не сохранённые).
- Токен не логировать; ошибки — через redact (если есть).
- verify: `backend/tests/` (проверка check по телу), preview — ввести новые креды, «Проверить» тестирует их до
  сохранения.

## Критерии готовности плана E

- Терминал SSL сворачиваемый, Домены на его месте, авто-разворот при задаче.
- График загрузки нод крупный и читаемый.
- ИИ-чат — отдельный раздел после «Автоматизации».
- «Проверить соединение» тестирует введённые в поля креды. `tsc` + pytest + preview.
