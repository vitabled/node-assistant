14-шаговый deploy-pipeline, Remnawave-клиент, xray-checker аналитику + subs-aggregator, инфра-биллинг,
установку панели/подписок (Волна 1 Ф3-Ф9), движок правил + MCP + ИИ-агент + синхронизацию + миграцию +
редактор Xray-профилей (Волна 2). Волна 3 — набор продуктовых доработок и заимствований из
экосистемы Remnawave (docs.rw) и `eGamesAPI/remnawave-reverse-proxy` (wiki.egam.es).

## Закреплённые решения (Alignment — в фоне не переспрашивать)

**Дэшборд/мониторинг (план A):**
- Мониторинг **включён по умолчанию**: `XrayCheckerConfig.enabled` дефолт → `True`; в lifespan **автостарт**
  контейнера, если есть `subscription_url` + доступен Docker; иначе тихо «не настроен» (без падений).
- Группировка «Xray uptime»: **двухуровневая — по подписке (верхний), внутри по странам** (страна парсится
  из имени ноды через `flagFor`/`COUNTRIES`; не извлеклась → «Прочее» внутри подписки).
- Баг «новая подписка видна только после Обновить / вторая не появилась» — **фикс**: чекер перечитывает
  `SUBSCRIPTION_URL` только по интервалу; при добавлении/включении/refresh подписки делаем **debounced
  reload чекера** (перезапуск контейнера) + `/refresh` агрегатора. Ошибки апстрима явно показываем.
- **«Server uptime»** — новый монитор: онлайн = **TCP-connect на порт (дефолт 443, фолбэк 22) + ICMP-ping**,
  статус up/slow/down (slow по порогу RTT). Источники: **ручные сервера (Название/Страна/IP/Примечание,
  CRUD add/edit/delete)** + **автоподтягивание задеплоенных нод** из `deploy_jobs` (read-only, помечены).
- 6b: селектор типа чекера в виджетах «Самые стабильные/быстрые» получает **server-uptime** как источник «другой».

**Деплой (план B):**
- 2a: «Установить Hysteria2» = **тумблер `install_hysteria2`**, гейтит существующий шаг 14 (`step_certbot_ssl`,
  лейбл «Hysteria2»). Реальный протокол Hysteria2 НЕ ставим (вариант А).
- 2b: в секции Remnanode — **2 горизонтальные вкладки «eGames» / «Vanilla»**. eGames = текущее поведение.
  **Vanilla** = установка по офиц. доке `remnawave/node:latest` (`network_mode: host`, `SECRET_KEY`, серты
  панели), **без домена и без маскировки** (жёстко выкинуты шаги 10/12); тумблеры **WARP и Hysteria2
  остаются**; нода регистрируется в Remnawave как обычно.
- 2c/E1/E3/E4 — заимствования из eGames-доки:
  - **E1 Docker registry-mirror + IPv6-fix** — тумблер, **для всех** деплоев (нода и панель), правит
    `/etc/docker/daemon.json` перед `docker compose up`.
  - **E3 Xray Torrent Blocker** — тумблер, **для всех**; новый управляемый компонент ноды (install/uninstall).
  - **E4 cookie/secret-gate nginx** — тумблер, **только для eGames-установки ноды** (маскировка хоста от сканеров).
  - E2 (WARP-routing по доменам) — **уже покрыт** установкой WARP, отдельно не делаем.

**Редакторы конфигов (план C):**
- 4a: заменить `<textarea>` в Шаблонах на **CodeMirror-редактор** из профилей (`JsonEditor.tsx`, подсветка
  JSON + подчёркивание ошибок через lint).
- 4b: добавить переменную **`$xhttp_path`** (= `xhttp_path` из формы деплоя) в подстановку Xray-config-шаблона
  при `create_config_profile`; в подсказке редактора перечислить все переменные.
- 5a: разрешить переменные `$domain`/`$xhttp_path`/`$name` в строковых полях **хост-шаблона**, подстановка в
  `step_create_hosts` с **shell-валидацией** (CLAUDE.md §5 forward-note). **Убрать селектор внутреннего/внешнего
  сквада** — сквад подтягивается автоматически при деплое хоста в Remnawave (union инбаундов, как уже делает
  `add_inbounds_to_internal_squad`).
- 10a: перенести «Профили» в сайдбаре **сразу после «Шаблонов»**.
- 10b: **вариант C — двусторонняя авто-синхронизация** профили↔шаблоны через общий источник данных (созданный
  шаблон виден как профиль и наоборот). `$`-плейсхолдеры шаблона допустимы в строковых полях визуального
  редактора (валидация по Xray-схеме их не роняет — down-rank до warning как уже сделано для enum).

**Панель/подписки (план D):**
- Новая «страница подписок» = **официальный бандл `remnawave/subscription-page`** (статический SPA), а не один
  HTML. Разворачивается Docker-сервисом `remnawave/subscription-page:latest`, подключается к панели через
  `.env`: `REMNAWAVE_PANEL_URL` + `REMNAWAVE_API_TOKEN` (токен создаётся в панели «Dashboard → API Tokens»),
  `APP_PORT=3010`, reverse-proxy (caddy/nginx); на панели — `SUB_PUBLIC_DOMAIN`. Механика:
  docs.rw/install/subscription-page/separate-server.
- 8a: при установке/после установки sub-page — **привязать к панели**: (1) к панели из `panel_jobs`, либо (2) к
  внешней панели по URL; node-assistant **создаёт API-токен в панели** и прописывает в `.env` sub-page.
- **Несколько sub-page на одну панель** (список, каждая со своим доменом/токеном).
- 9: переписать backend-редактор подписок под новый бандл-формат; добавить **загрузку/выбор архивов** (zip)
  со страницами подписок (кастомный фронт-бандл).

**Настройки/UX (план E):**
- 3a: окно вывода терминала в «Управление SSL» — **сворачиваемое** (по умолчанию свёрнуто, на его месте
  Домены; при старте новой задачи авто-разворачивается).
- 6a: «Загрузка нод во времени» — перерисовать как **крупный линейный график** (ось Y, сетка, ховер-тултип,
  подписи «сейчас/пик»).
- 11a: **перенести ИИ-чат** (`settings/AiChat.tsx`) в топ-левел сайдбара **сразу после «Автоматизация»**;
  конфиг MCP-сервера остаётся в Настройках.
- 11b: «Проверить соединение» Remnawave — проверять **значения из полей формы** (в теле запроса), ничего не
  сохраняя.
- 11c — **отменён** (перенос «Оптимизации ОС» не делаем).
