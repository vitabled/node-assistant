## Развилки, закреплённые пользователем (действуют на всю волну)

1. **Страница подписок:** baseline берём **стягиванием с живого контейнера** по SSH и кэшируем по digest образа;
   вариант = overlay (только изменённые файлы). Делаем **оба** уровня: сначала штатные
   `subscription-page-configs` через API панели, потом file-overlay.
2. **Ассистент:** CLIProxyAPI поднимаем **сами**, DooD-контейнером на `node-assistant-net` (как `xray-checker`
   и MCP). Это доделывает отложенную опц. Ф2 Плана J Волны 5.
3. **eGames:** **портируем конфиги** в наш `panel_pipeline` как `panel_variant`; их bash-установщик НЕ дёргаем.
   Для eGames-варианта nginx **контейнерный** (1:1 с апстримом, один редактируемый `/opt/remnawave/nginx.conf`);
   существующие caddy/nginx-варианты не трогаем.
4. **Перф:** **точечно по измерениям.** Ф1 плана F — зафиксировать базу; правим только найденное.
5. **Два уже сломанных места включены в волну** (см. ниже). Остальное найденное — в бэклог «later».

## Что уже сломано в проде (входит в волну)

- **Деплой страницы подписок ломает страницу.** Мы монтируем произвольный `index.html` поверх шаблона
  собранной Vite/React SPA, которую NestJS отдаёт через **EJS** (`<%- panelData %>`) — это тихо убивает
  единственный канал данных SPA. Наш стор/UI построены под *legacy Orion*, а не под образ, который мы реально
  разворачиваем. Плюс в новых версиях `REMNAWAVE_API_TOKEN` стал **обязательным** — без него контейнер падает.
  → План **E**, Ф1.
- **Тумблер Cookie-gate на ноде мёртв.** Поле `cookie_gate` есть в модели и переключатель есть в форме, но в
  пайплайне **ноль потребителей**; CLAUDE.md §9a при этом описывает его как работающий. → План **D**, Ф5.

Отдельно проверено вручную и подтверждено: **внутренний режим шлюза недостижим** — `AiConfig.gateway_internal`
читается в `ai_agent.py:208`, но сеттера нет нигде и в `AiConfigBody` его нет. → План **C**, Ф3.

## Бэклог «later» (найдено разведкой, сознательно вне волны)

- Мёртвый theme/motion-код: `Skeleton`/`.ni-skeleton`, `Stagger`, `StaggerItem`, `AnimatedNumber`, `.toast`
  (+3 override-блока), `.term`, `.bar-cell` — нигде не используются, но CLAUDE.md §2a описывает их как живые.
- Мёртвый rail-режим сайдбара: `Sidebar` принимает `collapsed`/`onToggle` и не использует; `App.tsx` хранит
  `sidebar_collapsed` в localStorage без визуального эффекта.
- Фильтрация «Топ пользователей» по скрытым нодам — нужен новый backend-параметр (`_top_users` группирует по
  username и теряет `node_uuid`).
- Скрытие на вкладке Дэшборд → **Xray uptime** (карта `hidden.checker` из Плана B переиспользуема).
- Авто-создание `REMNAWAVE_API_TOKEN` через `POST /api/tokens` — требует admin-логина панели (API-ключ на том
  контроллере запрещён).
- Caddy-паритет для eGames-варианта (v1 гейтится на nginx, Caddy → 422).
- selfsteal/decoy-режим, unix-сокет + `proxy_protocol`, порт 8443 у eGames — нужны, только когда `:443` занят
  Xray на том же боксе.
