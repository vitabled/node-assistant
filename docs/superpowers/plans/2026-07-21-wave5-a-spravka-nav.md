# Волна 5 · План A — Навигация «Справка» + «Карта хостингов» + слот «Библиотека»

> Frontend-only реорганизация навигации. Группа сайдбара **«Хостинги» → «Справка»**; пункт **«Карта»
> (`hostings-map`) → «Карта хостингов»**; пункт-список **«Хостинги» (`hostings-list`)** остаётся под именем
> «Хостинги», но уже внутри группы «Справка». В группу добавляется новый пункт **«Библиотека»** — здесь только
> nav-слот: новый `Tab` `'library'`, маршрут-заглушка в `App.tsx` и запись в `CRUMB`. Сам раздел «Библиотека»
> (заметки/знания + файловое хранилище + поиск/превью, вкл. backend) реализует
> `docs/superpowers/plans/2026-07-21-wave5-c-library.md` — id `'library'` согласован между планами.
> Затрагивает: `frontend/src/components/Sidebar.tsx` (label группы, label пункта `hostings-map`,
> `HOSTINGS_TABS`, добавление library-пункта, тип `Tab`), `frontend/src/App.tsx` (`CRUMB` + маршрут-заглушка),
> `frontend/src/components/Sidebar.test.tsx` (проверки новых лейблов). Backend НЕ трогается (раздел «Библиотека»
> и его API — целиком в Плане C). Ничего в 14-шаговом пайплайне и per-account сторах не меняется.

## Контекст (как есть)
- `Sidebar.tsx`: тип `Tab` — union строк; есть значения `"hostings-map" | "hostings-list"` (строки 10-17).
- `HOSTINGS_TABS` (строки 51-54): `{ tab: "hostings-map", label: "Карта", Icon: MapIcon }`,
  `{ tab: "hostings-list", label: "Хостинги", Icon: Server }`.
- Группа рендерится с заголовком-микролейблом `Хостинги` (строки 131-133): разделитель → `<p class="micro">Хостинги</p>`
  → `HOSTINGS_TABS.map(...)`. Заголовок группы — **литерал в JSX**, не берётся из массива.
- `App.tsx` `CRUMB: Record<Tab, [string, string]>` (строки 54-82): `"hostings-map": ["Хостинги", "Карта"]`,
  `"hostings-list": ["Хостинги", "Хостинги"]`. `CRUMB` также служит валидатором восстановления таба из
  localStorage (`if (stored && stored in CRUMB)`, строка 89) → **любой новый `Tab` ОБЯЗАН иметь запись в
  `CRUMB`**, иначе таб не переживёт перезагрузку.
- `App.tsx` маршруты (строки 223-224): `{tab === "hostings-map" && <HostingsMap/>}`,
  `{tab === "hostings-list" && <HostingsCatalog/>}`. Импорты `HostingsMap`/`HostingsCatalog` (строки 27-28).
- **Мобильный `BottomTabBar.tsx`**: собственный фиксированный список из 4 табов (dashboard/deploy/certs/traffic)
  + «Ещё» → открывает drawer. НЕ ссылается на hostings-лейблы → менять не нужно. Полная навигация на мобиле =
  тот же `Sidebar` с `drawer` пропом (наследует все лейблы автоматически).
- `Sidebar.test.tsx` (vitest+testing-library): проверяет наличие лейблов групп/пунктов (main, Remnawave, infra).
  Группу «Хостинги»/«Справка» сейчас НЕ проверяет.

## Развилки (закреплены)
- Заголовок группы = **«Справка»** (микролейбл в `Sidebar.tsx`); порядок пунктов: «Карта хостингов» →
  «Хостинги» → «Библиотека» (карта первой, как сейчас; библиотека последней — контентный раздел).
- Пункт `hostings-map` → **«Карта хостингов»**; `hostings-list` остаётся **«Хостинги»**; id табов НЕ меняются
  (`hostings-map`/`hostings-list` сохраняются — переименование только визуальное, не ломает localStorage-персист
  активного таба и per-account сторы хостингов).
- Новый пункт: `Tab = 'library'`, label «Библиотека», иконка — `BookOpen`/`Library` из lucide (согласовать с
  Планом C; по умолчанию `BookOpen`).
- **Маршрут-заглушка** (в этом плане): `{tab === "library" && <div>…«Раздел в разработке»…</div>}` — временно,
  до Плана C, который заменит заглушку реальным компонентом `Library`. Заглушка — минимальный theme-aware
  блок (CSS-var токены, без хардкода цветов), чтобы таб не рендерил пустоту.
- «В фоне не переспрашивать»: если План C уже смёржен и компонент `Library` существует — импортировать его
  вместо заглушки; иначе оставить заглушку.
